# Git 仓库瘦身清理

**时间:** 2026-06-14 23:47
**目标:** 解决 Git 操作极慢的问题

## 问题根因
- `.git` 目录膨胀到 667MB（占工作区 699MB 的 95%）
- 历史中包含 `output/pdf_study/` 的 37 个大 blob（PNG 图片，单个最大 2MB）
- 每次 git 操作都要扫描这个巨大的 `.git`，导致极慢

## 执行操作
1. 创建 `.gitignore`，加入 `output/`、`.DS_Store` 等规则
2. `git rm -r --cached output/` 从当前跟踪中移除 output 目录
3. `git filter-branch --index-filter 'git rm -rf --cached --ignore-unmatch output/'` 重写全部 26 个提交，从历史中彻底清除
4. 清理引用 `rm -rf .git/refs/original/` + `git reflog expire` + `git gc --prune=now --aggressive`
5. `git push --force` 同步到远程

## 结果
- `.git`: 667MB → 340KB（缩小约 2000 倍）
- 工作区: 699MB → 33MB
- output/ 目录仍在本地磁盘（32MB），但不再被 git 跟踪
- 以后 git 操作应该非常快
